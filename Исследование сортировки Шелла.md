# Задание: исследование сортировки Шелла с различными последовательностями шагов

## Цель
Исследовать влияние выбора последовательности шагов (коэффициентов) на производительность сортировки Шелла. Экспериментально сравнить быстродействие алгоритма при использовании разных последовательностей шагов.

## Задачи

1. Реализовать алгоритм сортировки Шелла.
2. Реализовать несколько стратегий выбора последовательности шагов:
    * последовательность Шелла;
    * последовательность Хиббарда;
    * последовательность Седжвика;
    * геометрическая последовательность с коэффициентом $K$.
3. Измерить время выполнения сортировки для каждого варианта на массивах разного размера.
4. Проанализировать и сравнить результаты.

### Последовательность выбора шагов

**1. Последовательность Шелла (оригинальная):**
* шаги: N/2, N/4,  N/8, &$\ldots, 1$;
* формула: $h_k = \lfloor N / 2^k \rfloor$, пока $h_k > 0$.

**2. Последовательность Хиббарда:**
* шаги: $\ldots, 31, 15, 7, 3, 1$;
* формула: $2^k - 1$, где $k = \ldots$, 3, 2, 1;
* максимальный шаг должен быть меньше размера массива.

**3. Последовательность Седжвика (упрощённая версия):**
* шаги: $\ldots, 109, 41, 19, 5, 1$;
* генерируется по формуле: $9 \times 4^k - 9 \times 2^k + 1$ или $4^k - 3 \times 2^k + 1$.

**4. Геометрическая последовательность:**
* шаги: $h, h/k, h/k^2, \ldots$, где $h$ — начальный шаг (размер массива / 2);
* округлять до целых, продолжать, пока шаг > 0.

### Измерение времени

* Использовать `time.perf_counter()` для точного замера времени.
* Для каждого сочетания (алгоритм + размер массива) проводить **5–10 замеров** и брать среднее значение.
* Измерять только время выполнения сортировки (без генерации массива).
**Размеры массивов**: выбрать 5–7 значений, равномерно распределённых в диапазоне от 100 до 10 000 (например: 100, 500, 1 000, 2 500, 5 000, 10 000, 50 000).
**Генерация массивов**: для каждого размера генерировать **случайный массив** целых чисел (например, в диапазоне от 0 до 1 000 000).


### Проверка корректности

* После каждого запуска проверять, что массив отсортирован по возрастанию.
* При ошибке выводить сообщение с указанием стратегии и размера массива.
